使用GitHub pages + Hexo搭建blog

前言

本文为本人搭建blog的过程记录。仅供参考。

一、GitHub创建个人仓库

登录到GitHub,如果没有GitHub帐号，则注册一个。点击GitHub中的New repository创建新仓库，仓库名应该为：${username}.github.io（${username}使用你的GitHub帐号名称代替），该仓库名为固定写法，且必须保持公开。

二、安装Git

安装

macOS可以通过安装Xcode命令行工具来完成git等相关工具的安装与配置
打开终端，执行以下命令即可：

xcode-select --install

其他平台可以在Git官网下载安装，并配置环境变量。

验证

在终端中输入以下命令：

git --version

如果出现以下信息，即为安装成功：

设置（可选）

ps.安装完成后，可以设置用户名与邮箱，添加ssh密钥到GitHub中，以便后续操作方便。
具体添加方式如下：

  1. 在终端中输入以下命令，设置用户名与密码： 
    # 设置用户名
git config --global user.name "${username}"
# 设置邮箱
git config --global user.email "${email}"
  2. 在终端输入以下命令，生成ssh密钥文件：
    ssh-keygen -t rsa -C "${email}"
    执行时可全部直接回车到下一步，不用设置密码（通行证）
  3. 添加ssh密钥文件至GitHub中
    在终端中执行以下命令：
    cat ~/.ssh/id_rsa.pub
    结果如下图，复制密钥信息
  4. 打开GitHub - settings - ssh and gpg kyes页面，点击new key，将之前复制的密钥信息粘贴至key对话框中，点击确定

三、安装NodeJS

安装

macos可以使用homebrew安装node，在终端执行以下命令： 

brew install node

如果没有安装homebrew，则先在终端执行以下命令安装homebrew：

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

其他平台可以在NodeJS官网下载安装，并配置环境变量。

验证

在终端输入以下命令：

node -v
npm -v

如果出现以下信息，即为安装成功：

四、安装Hexo与初始化

Hexo就是个人博客网站的框架，需要先创建一个空文件夹，这里命名为hexo，Hexo框架与以后你自己发布的网页都在这个文件夹中。创建好后，在终端中进入该路径。
在终端中输入以下命令安装hexo 

npm install -g hexo-cli

等待安装完成后，继续在终端中输入以下命令执行blog的初始化

hexo init

至此，网站初步搭建完毕。下面开始检测是否在部署成功
在终端输入以下命令，新建一篇文章，这里就生成默认标签的hello world

# 该命令为 hexo new 的简写
hexo n "hello world"

在终端输入以下命令，生成静态文件

# 该命令为 hexo generate 的简写
hexo g

执行完成后如下图：

静态文件生成后，执行以下命令，启动hexo服务： 

# 该命令为 hexo server 的简写
hexo s

执行完成后如下图：

打开浏览器，访问http://localhost:4000即可访问刚刚新建的hello world文章

hexo 相关的常用命令

# 安装Hexo
npm install hexo -g

# 升级Hexo
npm update hexo -g

# 初始化博客 folder未指定，则使用当前路径
hexo init [folder]

# 新建文章。hexo new 的简写。如果layout没有指定，则默认使用 _config.yml 中的 default_layout 参数代替
# 相关参数如下：
# -p, --path	自定义新文章的路径
#-r, --replace	如果存在同名文章，将其替换
#-s, --slug	文章的 Slug，作为新文章的文件名和发布后的 URL
hexo n [layout] <title>

# 生成静态文件。hexo generate 命令的简写。
# 相关参数如下：
# -d, --deploy	文件生成后立即部署网站
# -w, --watch	监视文件变动
# -b, --bail	生成过程中如果发生任何未处理的异常则抛出异常
# -f, --force	强制重新生成文件 Hexo 引入了差分机制，如果 public 目录存在，那么 hexo g 只会重新生成改动的文件。使用该参数的效果接近 hexo clean && hexo generate
# -c, --concurrency	最大同时生成文件的数量，默认无限制
hexo g

# 启动服务。hexo server 的简写。通常用于本地预览
# 相关参数如下
# -p, --port	重设端口
# -s, --static	只使用静态文件
# -l, --log	启动日记记录，使用覆盖记录格式
hexo s

# 部署。hexo deploy 的简写。
# 相关参数如下
# -g, --generate	部署之前预先生成静态文件
hexo d

# 清除缓存
hexo clean

五、发布文章

以上完成了hexo的本地部署与预览，然后需将生成的文章推送到GitHub pages网站上，让别人看到。

文章位置

hexo生成的文章，放在hexo根目录下的/source/_posts下， 如下图：

文章格式为.md，也可自行创建markdown文件放入其中。

推送插件

在hexo根目录下执行以下命令，安装GitHub推送插件

npm install hexo-deployer-git --save

hexo配置文件

在hexo根目录有_config.yml配置文件。该文件为站点的配置文件。在hexo/themes/具体主题目录下，有_config.yml配置文件，该文件为主题的配置文件。为了方便区分与修改，可以在hexo根目录下，新建主题配置文件。如果使用自带主题，则hexo根目录下已有对应的主题配置文件_config.landscape.yml。如果使用其他主题，则新建文件_config.${themeName}.yml，将hexo/themes/${themeName}下_config.yml中的内容复制进去。

打开hexo根目录下的_config.yml文件，如下图：

将hexo与GitHub关联起来，打开配置文件，在最底部，修改配置，参考如下：

配置修改并保存后，当执行部署命令hexo d时，会将当前本地生成的静态文件推送至GitHub pages。打开浏览器，在地址栏输入个人page的路径即可访问。路径名为https://${username}.github.io。比如我的博客。

六、更换主题

hexo默认的主题如果不喜欢，可以自行更换。主题传送门：Themes。下面为我现在使用的minimalism主题更换方式：

  1. 在主题网站中找到minimalism的地址。

  2. 在hexo根目录下打开终端。

  3. 输入以下命令，将主题下载下来，因为我的hexo也加入了版本控制，所以这里将主题添加为子模块。

    # 先进入到主题目录中
cd themes
git submodule add git@github.com:f-dong/hexo-theme-minimalism.git

# 或者直接
git submodule add git@github.com:f-dong/hexo-theme-minimalism.git themes/hexo-theme-minimalism

  4. 打开配置文件_config.yml将主题选项配置为刚刚下载的主题，如下图：

  5. 在hexo根目录下新建主题配置文件_config.hexo-theme-minimalism.yml。将hexo/themes/hexo-theme-minimalism中_config.yml中的内容复制进去。

关于git子模块，具体可以参考git子模块官方说明
以下为子模块常用命令(以添加的主题为例):

# 添加子模块
git submodule add git@github.com:f-dong/hexo-theme-minimalism.git

# clone时，将子模块一同clone
git clone --recurse-submodules git@github.com:FrozenBoar/blog.git

# 如果clone时，没有将子模块一起clone，则使用以下命令单独clone子模块
git submodule update --init --recursive

# 更新子模块
git pull --recurse-submodules

主题其他配置

站点配置文件_config.yml的其他修改

  • 修改url的值为你的网站主 URL（如：http://xxx.github.io）。
  • 修改两个per_page的分页条数值为6的倍数，如：12、18等，这样文章列表在各个屏幕下都能较好的显示。
  • 修改language的值为zh-CN，否则部分浏览器将弹出翻译窗口。

新建标签 tags 页

tags页是用来展示所有标签的页面, 如果在你的博客source目录下还没有tags/index.md文件, 那么你就需要新建一个, 命令如下:

hexo new page "tags"

编辑你刚刚新建的页面文件/source/tags/index.md，至少需要以下内容：

---
title: tags
date: 2022-04-26 22:59:30
type: "tags"
layout: "tags"
---

新建关于我 about 页

about页是用来展示关于我和我的博客信息的页面，如果在你的博客source目录下还没有about/index.md文件，那么你就需要新建一个，命令如下：

hexo new page "about"

编辑你刚刚新建的页面文件 /source/about/index.md，至少需要以下内容：

---
title: about
date: 2022-04-26 22:59:30
---

新建标签 categories 页(可选)

本主题内置了一个categories归档页, 用于显示分类下的所有文章, 如果在你的博客source目录下还没有tags/categories.md文件, 那么你就需要新建一个, 命令如下:

hexo new page "categories"

编辑你刚刚新建的页面文件/source/categories/index.md, 至少需要以下内容:

---
title: categories
date: 2022-04-26 22:59:30
type: "categories"
layout: "categories"
---

安装中文链接转拼音插件

如果文章名称是中文的，那么 Hexo 默认生成的永久链接也会有中文，这样不利于 SEO。我们可以用hexo-permalink-pinyin Hexo 插件使在生成文章时生成中文拼音的永久链接。
打开终端，进入hexo根目录，输入以下命令安装： 

npm i hexo-permalink-pinyin --save

代码高亮

从 Hexo5.0 版本开始自带了 prismjs 代码语法高亮的支持，本主题对此进行了改造支持。

如果你的博客中曾经安装过hexo-prism-plugin的插件，那么你须要执行

npm uninstall hexo-prism-plugin

来卸载掉它，否则生成的代码中会有&#123;和&#125;的转义字符。

然后，修改 Hexo 根目录下_config.yml文件中highlight.enable的值为false，并将prismjs.enable的值设置为true，主要配置如下：

highlight:
  enable: false
  line_number: true
  auto_detect: false
  tab_replace: ''
  wrap: true
  hljs: false
prismjs:
  enable: true
  preprocess: true
  line_number: true
  tab_replace: ''

本地图片显示的解决办法：

  1. 需要安装本地图片插件
    npm install hexo-asset-image --save
  2. 修改hexo根目录下/node_modules/hexo-asset-image/index.js文件第23行左右。将link.lastIndexOf('.')修改为link.lastIndexOf('/')即可。
    原文件如下：
    if(/.*\/index\.html$/.test(link)) {
  // when permalink is end with index.html, for example 2019/02/20/xxtitle/index.html
  // image in xxtitle/ will go to xxtitle/index/
  appendLink = 'index/';
  var endPos = link.lastIndexOf('/');
}
else {
  var endPos = link.lastIndexOf('.');
}
    修改后的文件如下：
    if(/.*\/index\.html$/.test(link)) {
  // when permalink is end with index.html, for example 2019/02/20/xxtitle/index.html
  // image in xxtitle/ will go to xxtitle/index/
  appendLink = 'index/';
  var endPos = link.lastIndexOf('/');
}
else {
  var endPos = link.lastIndexOf('/');
}

  3. 修改hexo目录下面的_config.yml文件，修改post_asset_folder的值为true。如下图：

    后续在hexo new "文章名"新建文章后，source/_posts/目录下面会生成一个和markdown文件同名的文件夹。
    将当前这篇文章中需要插入的图片放到这个文件夹中，再到 markdown 中引用该文件夹中的图片即可。

文章链接永久化方案

每次更改文章题目或者变更文章发布时间，在hexo默认设置下，文章链接都会改变，不利于搜索引擎收录，也不利于分享。唯一永久链接才是更好的选择。

  1. 安装链接持久化插件
    npm install hexo-abbrlink --save
  2. 修改站点配置文件_config.yml中的permalink，修改为以下： 
    permalink: posts/:abbrlink/ # “posts/” 可自行更换
  3. 添加abbrlink配置
    在站点配置_config.yml中添加以下内容：
    # abbrlink config
abbrlink:
  alg: crc32  # 算法：crc16(default) and crc32
  rep: hex    # 进制：dec(default) and hex
    可选模式有
    crc16 & hex
crc16 & dec
crc32 & hex
crc32 & dec

引用站内文章的方法

{% post_link bolg-name 点击这里查看这篇文章 %}

  • bolg-name是你的文章名称，不带.md。如果文章不存在，这段代码将会被直接忽略。
  • “点击这里查看这篇文章”是该链接的标题。如果置空，则自动提取文章的标题。
